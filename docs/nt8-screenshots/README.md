# NT8 screenshots (Flux V1 / PT3)

**Purpose:** Operator-captured evidence for `docs/nt8-backtest-methodology.md`.  
**Workflow:**

1. Commit PNGs here with **short descriptive filenames** (examples below).
2. After files land on `main`, ask the coding agent to **merge literal values into §§2–9** of `nt8-backtest-methodology.md`, mark **§11** checklist items complete, set document **Status** to *Complete* with date, and add a **Revision history** row that indexes this folder.
3. Redact account numbers or broker-specific secrets if needed; keep **numerical** settings (ticks, dollars) visible for M4 defaults.

**Suggested filenames (adjust if NT8 UI differs):**

| File (example) | Content |
|----------------|---------|
| `strategy-properties-flux-v1.png` | Strategy Properties (one representative live Flux V1 strategy). |
| `strategy-analyzer-general.png` | Strategy Analyzer — main / data / capital. |

| `strategy-analyzer-commission.png` | Commissions or linked template. |
| `instrument-manager-mnq.png` | MNQ (or backtest series) — tick size, point value. |
| `session-template-name.txt` | *Optional.* One-line file with session template name if not visible on other shots. |
